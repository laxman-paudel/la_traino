# AI_RULES — La Traino

## Project Identity

- **Name:** La Traino — A Personalized Physical Training System
- **Type:** BSc CSIT College Project
- **Goal:** Build a complete MVP that follows the approved project proposal exactly. Do not add enterprise or unnecessary features.

## Budget

$0. Every technology, library, deployment platform, and service must be free. No paid services are permitted.

## Approved Tech Stack

| Layer        | Technology                       |
|-------------|----------------------------------|
| Frontend    | React (Vite), React Router, Tailwind CSS |
| Backend     | Node.js, Express.js              |
| Database    | PostgreSQL                       |
| ORM         | Prisma                           |
| Authentication | Email & Password (JWT + bcrypt), Google OAuth |

## Hard Rules

1. **Stack lock** — Never change the tech stack without explicit approval.
2. **No placeholders** — Never generate placeholder or stub code. Every line must serve an actual feature.
3. **No over-engineering** — Build only what the proposal describes. Keep everything simple.
4. **Clean code** — Code must be clean, modular, and readable. Use meaningful file and variable names.
5. **Ask before deciding** — When making an important architectural choice, explain it briefly and ask for confirmation.
6. **One milestone at a time** — Complete the current milestone fully before moving on. Wait for approval before starting the next.
7. **Scope discipline** — Never add features absent from the proposal unless the user explicitly requests them.
8. **Free only** — Always prefer free and open-source solutions.
9. **Deployable for free** — The application must be deployable on free-tier services (Vercel/Netlify for frontend, Render/Railway for backend, Neon.tech for database).
10. **Ask first** — If unsure about an architectural decision, ask before proceeding.
11. **Authentication Requirements** — The application must support:
    - Register with Email
    - Login with Email
    - Login with Google OAuth
    - JWT-based session authentication
